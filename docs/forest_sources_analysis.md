# Анализ источников данных о породном составе лесов для России

Этот документ фиксирует разбор, какие источники дают нам классификацию
лесов по породам для **российской территории** — в частности для Ленобласти.
Сделано после того, как мы упёрлись в границу EEA39 у Copernicus HRL.

Главный вывод: **глобальные бесплатные продукты дают потолок
«хвойные / лиственные / смешанные»**. Чтобы получить реальные породы
(ель, сосна, берёза и т.д.) для России, нужны **российские научные
или кадастровые данные**.

---

## Ключевой факт: граница EEA39

Все «High Resolution Layer» продукты Copernicus Land Monitoring Service
(HRL DLT, HRL Tree Species, HRL TCD) заканчиваются на государственной
границе России. Проверено 2026-04-14: Map Viewer рендерит данные по
Финляндии, Эстонии, прибалтийским странам, но не по Ленобласти и
Карелии. Это жёсткое лицензионное/техническое ограничение EEA39
(страны EEA + UK + Западные Балканы), не баг просмотрщика.

**Что это значит:** наш свеженаписанный `CopernicusForestSource`
(реализация есть, работает на синтетике) для Ленобласти бесполезен
напрямую. Нужны альтернативы.

---

## Три уровня точности, которые реально достижимы для России

### Уровень 1 — хвойные/лиственные/смешанные

Потолок **глобальных бесплатных продуктов**. Эти источники покрывают
всю Россию, регистрация минимальная или не нужна, интеграция простая
(тот же GeoTIFF + class_map).

| Продукт | Разрешение | Классы | Источник |
|---|---|---|---|
| **Copernicus Global Land Cover (CGLS-LC100)** | 100 м | `closed_forest_evergreen_needle_leaf`, `closed_forest_deciduous_broad_leaf`, `closed_forest_mixed` + fraction cover | https://lcviewer.vito.be/ |
| **ESA CCI Land Cover** | 300 м | Классы 50/60/70/80 (BL-EV / BL-DC / NL-EV / NL-DC) | https://climate.esa.int/en/projects/land-cover/ |
| **ESA WorldCover 2021** | 10 м | Только «Tree cover» одним классом | https://esa-worldcover.org |
| **MODIS Land Cover MCD12Q1** | 500 м | Evergreen/deciduous × needle/broad | GFW, earthengine |

**CGLS-LC100 @ 100м** — лучший вариант этого уровня для нас. Покрывает
Россию, есть типы леса (не только «лес/не лес»).

**Чего не дают:** сосну vs ель vs лиственницу — всё это попадает в
«хвойные».

### Уровень 2 — группы пород (ель, сосна, лиственница, берёза)

**TerraNorte RLC** — продукт **Института космических исследований РАН**
(группа С.А. Барталёва, ИКИ РАН). Единственный публичный научный
продукт, который **специально разработан для лесов России** и доступен
без кадастровой бюрократии.

- **Разрешение:** 230 м (основан на временных рядах MODIS Terra/Aqua)
- **Охват:** вся Россия, ежегодные обновления
- **Классы** (примерно, уточнить по актуальным метаданным):
  - Тёмнохвойные (ель, пихта) → наш slug `spruce` или `fir`
  - Светлохвойные вечнозелёные (сосна) → `pine`
  - Светлохвойные листопадные (лиственница) → `larch`
  - Широколиственные (дуб, липа, клён) → `oak`/`linden`/`maple` (без уточнения)
  - Мелколиственные (берёза, осина, ольха) → `birch`/`aspen`/`alder` (без уточнения)
  - Смешанные, молодняки, гари
- **Доступ:** http://terranorte.iki.rssi.ru/
- **Формат:** GeoTIFF, наш `CopernicusForestSource` (переименуем в
  `RasterForestSource`) переварит без изменений, только другой class_map

**Что мы получим:** для среднего лесного полигона Ленобласти 5–10 га =
1000–2000 пикселей TerraNorte внутри — вполне хватит, чтобы посчитать
доминанту и грубую смесь. Качество — научное, есть публикации в
Remote Sensing of Environment.

**Ограничения:**
- 230 м грубее чем HRL 10 м — границы между породами размыты
- Не различает сосну и лиственницу отдельно в некоторых годах (зависит
  от версии продукта)
- Может требовать регистрации

**Это тот уровень, на который мы нацелились в шаге 2 плана.**

### Уровень 3 — реальные виды со смесью (ель 60%, сосна 30%, берёза 10%)

**Rosleshoz ВЛС / ФГИС ЛК** — официальный лесной кадастр России.
Содержит **породный состав каждого выдела** в виде формулы
«6Е3С1Б» (6 частей ель, 3 сосна, 1 берёза).

- **Покрытие:** ~80–90% лесов Ленобласти (Гослесфонд, исключает
  частные и городские леса)
- **Разрешение:** выдел (обычно 1–50 га), границы геометрически известны
- **Доступ:**
  - https://lk.rosleshoz.gov.ru (публичный просмотрщик с картой)
  - https://lesegais.ru (ФГИС ЛК)
- **Публичного API нет** — нужен скрапинг tile-сервера ИЛИ запрос в
  лесничество ИЛИ региональный геопортал.
- **Структура данных идеально ложится на нашу схему** —
  `forest_polygon.species_composition` JSONB уже готов хранить
  `{"spruce": 0.6, "pine": 0.3, "birch": 0.1}`.

**Код готов, данные — user-supplied.** Что мы сделали 2026-04-14:

1. **Парсер формулы** `parse_species_formula("6Е3С1Б")` →
   `{"spruce": 0.6, "pine": 0.3, "birch": 0.1}`. Поддерживает 20+
   сокращений пород, trace через `+` и `ед.`, unmapped-виды
   (ива/тополь/граб/бук — нет своих slug'ов, не ломают композицию).
   См. [../services/geodata/src/geodata/sources/rosleshoz/formula.py](../services/geodata/src/geodata/sources/rosleshoz/formula.py).

2. **`RosleshozForestSource`** читает любой векторный файл через pyogrio
   (GeoJSON/Shapefile/GPKG/FlatGeobuf/GDB), автоопределяет поле с
   формулой, перепроецирует в WGS84, считает площади в EPSG:3035,
   выдаёт `NormalizedForestPolygon` с `confidence=0.95` и реальной
   `species_composition`.

3. **Миграция 010** регистрирует rosleshoz в `forest_source` с
   приоритетом 60 и переписывает `forest_unified` как полный каскад
   `rosleshoz(60) > copernicus(50) > terranorte(45) > osm(10)`.

4. **CLI**: `python pipelines/ingest_forest.py --source rosleshoz
   --region lenoblast --rosleshoz-file data/rosleshoz/...`.

5. **Документация**: [rosleshoz_download.md](rosleshoz_download.md) —
   пять путей получения данных (ФГИС ЛК, региональный портал,
   запрос в лесничество, open data, неофициальные сборки).

**Что остаётся user'у:** достать собственно векторный файл с формулами.
Это главный блокер — публичного API не существует, и ни один из
возможных путей не быстрый. Закладывайся на 1-4 недели.

Когда файл появится — всё остальное работает автоматически, включая
перегенерацию тайлов и обновление попапа на карте.

## Альтернативы, которые мы НЕ стали рассматривать (и почему)

- **Global Forest Watch / Hansen Global Forest Change** — про обезлесение,
  без классификации пород. Полезно для «где вообще лес», у нас это уже
  покрыто OSM.
- **Primary forests / Intact forest landscapes (IFL)** — бинарная маска
  «нетронутый лес», без пород.
- **Tropical Tree Cover (WRI)** — только тропики, не наш регион.
- **Sentinel-2 сезонный NDVI (DIY)** — можно сделать свой classifier
  «вечнозелёные vs листопадные» по разности NDVI лето/зима. Требует
  скачивания сырых S2-тайлов (CDSE даёт их на Россию), нескольких дней
  работы, и всё равно не даёт более чем «кониферы / декидуалы». То же
  что Уровень 1, только своими руками.
- **GEDI / Global Canopy Height** — высота крон, не породы.
- **Bartalev 2014 Russia Forest Cover, Shepashenko/IIASA Russian Forests** —
  отдельные научные датасеты, часть легла в основу TerraNorte. Отдельно
  качать нет смысла.

## Итоговый план действий

1. **Шаг 1 — OSM name heuristic (30 минут).** На существующих 47 000
   лесных полигонах запустить парсер имён: «Сосновый бор» → pine,
   «Ельник» → spruce, «Берёзовая роща» → birch, «Осинник» → aspen,
   «Дубрава» → oak. Должно снять с 88% unknown до ~60–70%. Бесплатно,
   быстро, результат прямо сегодня. **Чистый `forest_polygon` upsert,
   source='osm_heuristic' или обновление существующих OSM-строк.**

2. **Шаг 2 — TerraNorte RLC как новый ForestSource (1–2 дня).**
   Регистрация на http://terranorte.iki.rssi.ru/, скачивание растра,
   маппинг классов, ingest через новый `TerraNorteForestSource`
   (наследник от `CopernicusForestSource`), приоритет 45 в
   `forest_unified`. **Это цель текущей сессии после анализа.**

3. **Шаг 3 — Rosleshoz ВЛС scraping (долгосрок, отдельный
   спринт).** Скрапер, парсер формул, импорт выделов. Приоритет 60.

Архитектура `forest_source` + `forest_unified` это спокойно
переваривает: разные источники сосуществуют в одной таблице,
view выбирает приоритетный. Миграция = одна SQL-строчка на каждый
новый источник.

## Обновления CLMS коду

Файл `services/geodata/src/geodata/sources/copernicus.py` писался
специально универсальным — читает любой GeoTIFF с классификацией,
маппинг задаётся через `CopernicusConfig.class_map`. При переходе
на TerraNorte его **не надо переписывать**, только:

1. Унаследовать `TerraNorteForestSource` с `source_code="terranorte"`.
2. Положить свой `class_map` в default config.
3. Зарегистрировать в `sources/__init__.py::get_source`.

По сути, `CopernicusForestSource` — это уже `RasterForestSource` с
неправильным именем. Можно честно переименовать при рефакторинге.

## Контрольные точки для проверки при возврате к задаче

- **Граница EEA39** — проверить, не сменила ли CLMS лицензионную
  политику. Маловероятно, но раз в полгода стоит глянуть
  https://land.copernicus.eu/en/data-policy
- **TerraNorte** — возможно появится новая версия с лучшим разрешением.
  Мониторить http://terranorte.iki.rssi.ru/
- **Rosleshoz API** — иногда открывают публичные data-порталы. Следить
  за https://rosleshoz.gov.ru/opendata
