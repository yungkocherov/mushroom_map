# Идеи по наполнению сервиса

_Составлено: 2026-04-15_

Текущее состояние: лесные полигоны Rosleshoz/ФГИС ЛК (бонитет, запас, возраст), водоохранные зоны, теоретические и эмпирические виды грибов (ВКонтакте).

---

## Данные из ФГИС ЛК (тайлы уже есть, нужна разработка)

### Вырубки и гари
- **Источник:** слой `SPECIAL_CONDITION_AREA` в тайлах ФГИС ЛК
- **Что даёт:** категории «вырубка», «горельник», «молодняк после рубки». На 3–7-летних вырубках массово растут подосиновики, маслята, опята — отдельная экология по сравнению со взрослым лесом.
- **Сложность:** средняя — нужен новый слой тайлов + цветовая схема

### Защитные леса
- **Источник:** слой `PROTECTIVE_FOREST` в тайлах ФГИС ЛК
- **Что даёт:** запретные полосы, нерестоохранные леса, городские леса. Часть из них имеет ограничения на посещение и сбор.
- **Сложность:** средняя

## Внешние источники (открытые данные)

### ~~ООПТ — особо охраняемые природные территории~~ ✅ ГОТОВО (нет данных)
- **Источник:** opendata.gov.ru (Минприроды), портал ООПТ России (oopt.aari.ru), Geoportal.ru
- **Что даёт:** заповедники, заказники, национальные парки. Сбор грибов там либо запрещён, либо ограничен — прямая юридическая ценность. Пользователь видит красный полигон и понимает, что зайти нельзя.
- **Статус:** пайплайн `ingest_oopt.py` + `build_oopt_tiles.py` готовы, кнопка в UI есть. Нужно скачать GeoJSON с oopt.aari.ru (см. `docs/oopt_download.md`) и запустить пайплайн.

### Болота
- **Источник:** OpenStreetMap (`natural=wetland`, `wetland=bog`) или ФГИС ЛК (`SWAMP_AREA`)
- **Что даёт:** критически важно для безопасности — человек заходит в лес и не видит впереди топь. Также другая микология: клюква, морошка, специфические виды.
- **Сложность:** низкая (OSM данные уже могут быть в БД)
- **Приоритет: высокий**

### ~~Лесные дороги и просеки~~ ✅ ГОТОВО (нет данных)
- **Источник:** OpenStreetMap (`highway=track`, `highway=path`, `man_made=cutline`)
- **Что даёт:** грибники ходят по просекам и лесным дорогам. Показывать только на z12+.
- **Статус:** пайплайн `ingest_osm_roads.py` + `build_roads_tiles.py` готовы, кнопка в UI есть. Нужно скачать PBF с Geofabrik (см. `docs/osm_roads_download.md`) и запустить пайплайн.

### Рельеф (хиллшейдинг)
- **Источник:** SRTM / Copernicus DEM (30 м, открытые)
- **Что даёт:** карта становится читабельнее — видны холмы, ложбины. Низины = сырость = подберёзовики; склоны = дренаж = рыжики, грузди.
- **Сложность:** средняя — нужен растровый тайлсет

## Улучшение данных наблюдений ВКонтакте

### Тепловая карта находок (H3 слой)
- **Что даёт:** пользователь сразу видит «горячие точки» без клика по каждому полигону. Плотность наблюдений на гексагональной сетке H3 (уже есть в `observation_h3_species_stats`).
- **Сложность:** средняя — данные есть, нужен фронт-слой с color ramp

### Временной фильтр наблюдений
- **Что даёт:** фильтр «последние 2–3 года» резко повышает релевантность: лес может смениться после вырубки или пожара.
- **Сложность:** средняя — нужен UI-слайдер + параметр в API

### Лента последних находок
- **Что даёт:** «3 дня назад в этом районе нашли белые» — самая востребованная фича у грибников. Показать в сайдбаре для текущего viewport.
- **Требует:** дата поста хранится в наблюдениях ВК, нужен API-endpoint `/observations/recent?bbox=`
- **Сложность:** средняя
- **Приоритет: высокий**

## Практические слои

### Точки доступа в лес
- **Источник:** OSM — автобусные остановки (`highway=bus_stop`) + парковки у леса (`amenity=parking`)
- **Что даёт:** откуда заходить без машины / где парковаться. Показывать только на z11+.
- **Сложность:** низкая
