# Реестр видов: что считается грибом

В системе живёт замороженный словарь из **18 видов** (плюс 3 ягоды, на всякий случай). Этот список — контракт: на нём держится и ML-классификация фото, и матрица сопоставления с типами лесов, и подсветка слоёв на карте, и страницы вида в каталоге.

Менять словарь можно только в одну сторону — добавлять новые виды, не удалять и не переименовывать существующие. Слишком много кода завязано на slug'и (`boletus-edulis`, `cantharellus-cibarius`, ...).

## Что хранится

Таблица `species` в PostgreSQL содержит для каждого вида:

- **slug** — машинный идентификатор (`boletus-edulis`). Стабильный, ASCII, не меняется.
- **name_ru**, **name_lat**, **synonyms** — названия для отображения и поиска.
- **edibility** — одно из: `edible` / `conditionally_edible` / `inedible` / `toxic` / `deadly`.
- **season_months** — массив месяцев плодоношения (`[7, 8, 9, 10]` для белого).
- **description** — короткое прозрачное описание для страницы вида.
- **photo_url**, **wiki_url** — ссылки на основное фото (Wikimedia Commons) и Wikipedia-статью. Фото мы будем подгружать в Phase 2 — пока поле пустое.
- **red_book** — флаг краснокнижности (на территории РФ).

## Привязка к лесу

Отдельная таблица `species_forest_affinity(species_id, forest_type, affinity)` — матрица «насколько вид характерен для леса данного типа», 0..1. Типы леса — фиксированный словарь: `pine`, `spruce`, `birch`, `aspen`, `oak`, `mixed_coniferous`, `mixed_broadleaved`, `mixed`, `unknown`.

Например, белый гриб (`boletus-edulis`) имеет:
- `spruce: 0.9` — еловая форма
- `oak: 0.7` — дубовая
- `birch: 0.5` — берёзовая
- `pine: 0.3` — на сосне реже встречается белый, чаще — pine_bolete

Эти значения — экспертно-curated, лежат в `db/seeds/species_registry.yaml` и подгружаются через `species_registry.loader`. Они **не** обучены на данных — это сводка из ботанической литературы, которую можно править вручную при обнаружении ошибки.

В endpoint'е `/api/forest/at` для каждой точки выдаются «теоретические» виды — те, у кого `affinity` для доминирующей породы выше порога 0.4. На страницах видов (Phase 2) мы будем рисовать affinity-бары: пользователь сразу видит, в каком лесу искать.

## Группировка для классификатора

VK-классификатор в `pipelines/ingest_vk.py` оперирует не slug'ами, а более крупными «группами» — потому что qwen3.5-vl визуально не различает Leccinum aurantiacum и L. versipelle (два подосиновика), но человек на фото уверенно отличает «болет с оранжевой шляпкой» от «болета с серо-коричневой».

Маппинг GROUP_TO_SLUGS:

| Ключ из промта | Slug'и в species |
| --- | --- |
| `porcini` | `boletus-edulis` |
| `pine_bolete` | `boletus-edulis` (тот же slug — разделение только для статистики) |
| `aspen_bolete` | `leccinum-aurantiacum`, `leccinum-versipelle` |
| `birch_bolete` | `leccinum-scabrum` |
| `mokhovik` | `xerocomus-subtomentosus` |
| `chanterelle` | `cantharellus-cibarius`, `craterellus-tubaeformis` |
| `saffron_milkcap` | `lactarius-deliciosus` |
| `white_milkcap` | `lactarius-resimus` |
| `woolly_milkcap` | `lactarius-torminosus` |
| `spring_mushroom` | `morchella-esculenta`, `verpa-bohemica`, `gyromitra-esculenta` |
| `honey_fungus` | `armillaria-mellea`, `kuehneromyces-mutabilis` |
| `oyster` | `pleurotus-ostreatus` |
| `russula` | `russula-vesca` |
| `fly_agaric` | `amanita-muscaria` |

Когда классификатор видит «pine_bolete», в species-таблицу попадает та же запись `boletus-edulis`, что и для «porcini» — мы храним факт «нашли белый гриб». Различие хранится в JSONB-поле `vk_post.photo_species` для аналитики по тонам шляпок, но в финальном агрегате по виду они объединены.

## Чего мы намеренно НЕ показываем

1. **Числовую вероятность съедобности.** Только дискретный класс из пяти. Любая цифра «съедобность 87%» — это либо средняя по больнице (бесполезно), либо ложная уверенность (опасно). Если вид условно-съедобный — пользователь обязан прочитать описание.

2. **Региональные вариации сезона.** `season_months` глобальное; в реальности белый под Лугой и под Тихвином может пойти с разницей в неделю-две. Эта поправка появится с прогноз-моделью (Phase 3+), отдельной фичей региональной фенологии — пока мы на это не претендуем.

3. **«Лучшие места» для конкретного вида в публичном виде.** Мы показываем районы, где вид чаще всего фигурирует в VK-постах, — но не геокоординаты конкретных полян. Пользователь, который хочет «куда ехать», получает район; куда внутри района — это его собственная разведка.

## Расширение реестра

Чтобы добавить новый вид:

1. Дописать запись в `db/seeds/species_registry.yaml` (slug, name_ru, name_lat, edibility, season_months, description, forest_affinities).
2. Прогнать `make db-seed` — UPSERT по slug, ничего не ломает.
3. Если вид должен попадать в VK-классификатор — добавить ключ в `CLASSIFY_SCHEMA` enum в `pipelines/ingest_vk.py`, описать диагностические признаки в текущем `vk_classify_v{N}.txt`, бампнуть `PHOTO_PROMPT_VERSION`. Photos-stage перезапустится по всем постам с новой версией.
4. Если у вида есть фото — добавить URL Wikimedia Commons в `species.photo_url` (Phase 2 это будет автоматизировано отдельным скриптом).
